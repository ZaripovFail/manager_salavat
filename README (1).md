# МебельДан — цифровые визитки

Статический сайт для GitHub Pages. Одна папка = одна визитка менеджера.

## Структура

```
/style.css          ← общий стиль для всех визиток (меняешь тут — меняется у всех)
/fazliev/index.html ← визитка Салавата Фазлиева
```

## Как добавить нового менеджера

1. Скопируй папку `fazliev` и переименуй (например `ivanov`).
2. В `ivanov/index.html` замени:
   - `<title>` и монограмму (инициалы в `.monogram`)
   - имя (`.name`) и должность (`.role-line`)
   - ссылку `tel:` и номер в блоке `tel="TYPE=CELL"` в самом низу (JS для vCard)
   - ссылки Telegram / WhatsApp / МАХ / Instagram / канал
   - ссылки на Яндекс.Карты и 2ГИС, если адрес другой
   - имя файла в `a.download = '...'` в скрипте
3. Пуш в репозиторий — страница станет доступна по адресу
   `имя-репозитория.github.io/ivanov/`

## Публикация на GitHub Pages

1. Создай репозиторий, залей туда содержимое этой папки.
2. В настройках репозитория → Pages → Source → выбери ветку `main` и папку `/root`.
3. Через пару минут сайт будет доступен по адресу вида
   `https://<username>.github.io/<repo>/fazliev/`.
4. Для QR-кода на визитке или в мессенджерах — сгенерируй QR на этот URL
   любым онлайн-генератором (например qr-code-generator.com) и распечатай.

## Что можно доработать дальше

- Свой домен вместо `github.io` (подключается в настройках Pages).
- Общий `index.html` в корне со списком всех менеджеров (пригодится,
  когда карточек станет много и захочется единой точки входа).
- GitHub Actions, который сам собирает папки менеджеров из одного
  JSON-файла — стоит делать, когда шаблон уже устоится и не будет
  часто меняться в дизайне.
